# KEYWORDS — Google Ads Pilot, הסעות אבי ורדי
_On what searches the ads show. Hebrew copy (Israeli audience); structure in English._
_כל התוכן טיוטה — לעריכה לפני שימוש. שום דבר כאן לא נוגע בחשבון פרסום._

**Business:** הסעות אבי ורדי — נהג פרטי, אוטובוס אחד 56 מקומות. אזור שירות: ירושלים, גוש עציון,
אלעזר, אפרת והסביבה (יעדים בכל הארץ). המרה = **שיחת טלפון**, לא קליק.
**Match strategy:** start Phrase + Exact only (no Broad) so a ₪25/day budget buys intent, not noise.
Add Broad-match test only after a keyword proves profitable.

---

## Ad Group Structure (one Search campaign, 3 ad groups)
| Ad group | Intent | Primary keywords |
|---|---|---|
| נתב"ג / שדה תעופה | נסיעה לטיסה, מיידי | הסעה לנתב"ג, אוטובוס לנתב"ג |
| חתונות ואירועים | הסעת אורחים, מתוכנן | הסעות לחתונה, אוטובוס לאירוע |
| טיולים והשכרת אוטובוס | טיול/השכרה, רחב | השכרת אוטובוס עם נהג, אוטובוס לטיול |

Split rationale: intent + price band differ per group, so each gets its own keywords, ads, and
(later) its own cost-per-call read. Airport = highest frequency; weddings = highest value; tours =
broadest. Keep them separate so you can pause the weakest without touching the others.

## Campaign-name → lead-log handshake (do NOT skip)
Every campaign/ad-group name you create in Google Ads must be **dictated verbatim** to the lead log
(`/lead` → `leads.js`). Attribution is **exact-match** — a name that doesn't match `ads_daily`
mis-attributes the money in the ROAS view. When dad gets a call, he logs source+campaign; that call
is how "which search worked" becomes shekels. Suggested canonical names:
- `avivardi-search-natbag` (נתב"ג)
- `avivardi-search-weddings` (חתונות ואירועים)
- `avivardi-search-tours` (טיולים והשכרה)
